# AI 智能膳食管理系統

> 結合生理數據計算與生成式 AI 推論，為使用者提供精準、即時且個人化的飲食管理方案。
> 作者：吳翊愷

## 🚀 快速開始

### 方法一：直接開啟（最簡單）

雙擊 `index.html` 即可在瀏覽器中開啟使用。

```
AI智能膳食管理系統/
├── index.html          ← 雙擊這個檔案
├── styles.css
├── app.js
├── ai-engine.js
└── food-database.js
```

### 方法二：本機伺服器（推薦，可避免 CORS 問題）

開啟終端機，進入專案資料夾：

```bash
cd "/Users/kaiwu/AI智能膳食管理系統"
python3 -m http.server 8000
```

然後瀏覽器打開 `http://localhost:8000`

## ✨ 功能總覽

| 分頁 | 功能 |
|------|------|
| **個人資料** | 輸入性別、年齡、身高、體重、活動量、慢性病史與健康目標 |
| **AI 建議** | 取得專業個人化飲食策略報告（每日熱量、三大營養素、紅綠燈食物、實踐撇步） |
| **食物風險** | 輸入任何食物，依您的慢性病史評估風險等級 |
| **一日菜單** | 自動生成符合您熱量與疾病限制的早午晚餐參考菜單 |

### 內建臨床知識庫

支援以下 9 種慢性病的多病共管：

- 糖尿病（ADA / 台灣糖尿病學會建議）
- 高血壓（DASH 得舒飲食）
- 慢性腎臟病（低蛋白、低鈉、低磷、低鉀）
- 高血脂（地中海飲食）
- 痛風（低普林飲食）
- 心血管疾病（地中海 + DASH）
- 脂肪肝（減重 + 限糖）
- 骨質疏鬆（高鈣 + 維生素 D）
- 貧血（高鐵 + 維生素 C）

### 計算公式

- **BMI** = 體重(kg) / 身高(m)²
- **BMR** = Mifflin-St Jeor 公式（目前臨床公認最準確）
- **TDEE** = BMR × 活動係數（1.2 / 1.375 / 1.55 / 1.725 / 1.9）

## 🤖 啟用本地 AI（可選）

預設使用「內建推論引擎」（規則式 + 臨床知識庫），完全離線可用。

若希望取得更自然、富有彈性的對話式建議，可安裝 [Ollama](https://ollama.com) 並啟用本地 LLM：

### 1. 安裝 Ollama

```bash
# macOS
brew install ollama
# 或從官網下載：https://ollama.com/download
```

### 2. 啟動服務並下載模型

```bash
ollama serve              # 啟動服務（保持執行）

# 另開終端機，下載一個模型（推薦其中之一）
ollama pull llama3.1      # Meta Llama 3.1 (8B)
ollama pull qwen2.5       # 阿里通義千問（中文表現佳）
ollama pull gemma3        # Google Gemma 3
```

### 3. 在 App 內啟用

1. 點擊側邊欄右下角的齒輪按鈕
2. 開啟「使用本地 AI (Ollama)」
3. 確認模型名稱（如 `qwen2.5`）
4. 點擊「測試連線」確認成功
5. 回到「個人資料」分頁，重新生成建議即可

啟用後，AI 建議的「開頭問候段落」會由本地 LLM 動態生成，更個人化、更人性化。

## ⌨️ 鍵盤捷徑

- `⌘/Ctrl + Enter` — 生成個人化建議
- `ESC` — 關閉設定彈窗
